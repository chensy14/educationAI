작성일: 2026-03-27

언어: 한국어

## 1. 프로젝트 한 줄 요약

초등교사를 돕는 AI 교육 소프트웨어를 기획 중이다.  
핵심 목표는 **교사의 실제 귀찮은 업무를 줄이면서, 동시에 학생 학습에도 직접 도움이 되는 제품**을 만드는 것이다.

초기에는 `교사 업무 자동화`에 치우쳤으나, 현재 방향은 다음으로 정리되었다.

> **교사가 학년/교과/단원/출판사/난이도 등을 선택하면,  
> 교육과정 정합형 문제·해설·보충학습·수업용 PPT를 생성해 주는 서비스**

단, **교과서 원문/교사용 폐쇄 자료를 통째로 수집하는 방식은 현실적으로 어렵고**,  
현재 가장 현실적인 방향은 아래 구조다.

- 공개 교육과정/성취기준/메타데이터 활용
- 오픈 라이선스 자산 활용
- 문제/해설/PPT 본문은 자체 제작
- 필요 시 교사 업로드(BYOC) 또는 향후 제휴 확장

## 2. 지금까지 대화에서 확정된 핵심 결론

### 2.1 무엇을 만들지
현재 가장 유력한 코어 아이디어는 다음 둘 중 하나였고, 지금은 **둘의 결합형**으로 정리된 상태다.

1. **AI 형성평가 루프**
   - 성취기준/단원 기반 문항 생성
   - 학생 풀이 분석
   - 오답 해설
   - 유사문항 재도전
   - 교사용 오개념 분석
   - 재수업 자료 생성

2. **교사용 올인원 수업팩**
   - 단원/차시/목표 기반 PPT 초안
   - 활동지
   - 형성평가
   - 숙제/보충자료
   - 학습결과 기반 다음 차시 보강 자료

### 2.2 현재 추천 포지셔닝
범용 PPT 생성기보다는 아래가 더 강하다고 판단했다.

> **“교육과정 정합형 AI 문제·해설·재수업/PPT 생성 도구”**

즉:
- 그냥 예쁜 슬라이드를 만드는 AI가 아니라
- 교사의 실제 수업/평가 플로우에 맞게
- `문항 → 해설 → 오개념 분석 → 재수업 자료`까지 이어지는 도구

### 2.3 왜 범용 PPT AI만으로 부족한가
실제 교사와의 대화 및 자료 조사 기준, 교사가 원하는 것은 단순 PPT 생성이 아니라 아래에 가까움.

- 수업에 바로 쓸 수 있는 **검증된 교과연계 자료 묶음**
- 자료를 찾고 골라서 고치는 시간 절감
- 교과/단원/수준에 맞는 자료
- 활동지/퀴즈/보충문제까지 연결된 패키지

즉 교사가 말한 `올인원 PPT 자료 사이트/소프트웨어`는 실제로는 슬라이드 파일 하나보다  
**교과 정합형 수업자료 패키지**를 의미할 가능성이 높다.

## 3. 실제 교사 pain 관련 정리

### 3.1 교사 문제의 큰 축
지금까지의 조사/대화에서, 초등교사 pain은 대략 아래로 정리되었다.

- 수업 준비 자료 제작 부담
- 단원별 문항/평가/피드백 준비 부담
- 오답 분석 및 재수업 자료 준비 부담
- 학부모 소통/정리/문서화 부담
- 생활지도/상담/기록 부담

### 3.2 교육 소프트웨어로서 제일 맞는 pain
행정 자동화보다 **수업/평가/피드백 주변의 반복 작업**이 더 적합하다고 정리되었다.  
이유:

- 학생 학습효과와 직접 연결됨
- 교사 pain도 큼
- 공교육 예산 설득 논리가 있음
- 범용 AI보다 교사용 커스터마이징 여지가 큼

### 3.3 수업자료(PPT)는 필수인가
결론:
- **항상 필수는 아님**
- 하지만 매우 자주 쓰이는 도구일 가능성이 큼
- `있으면 좋은데, 매번 만들기 귀찮다`는 니즈가 존재함
- 특수교사도 “필수는 아니지만 항상 만들 수는 없다”는 반응이 있었음

즉 PPT는 `must-have`라기보다 **high-frequency pain 영역**에 가깝다.

## 4. 가장 중요한 조사 결과: 공개 자료로 무엇이 가능한가

사용자는 가장 중요하게 아래를 확인하고 싶어 했다.

> “실제로 학년/과목/단원 기준으로 교육자료 소스가 공개되어 있느냐?  
> 그걸 기반으로 문제나 PPT를 만들 수 있느냐?”

이 부분은 상당히 오해가 있었고, 검증 결과는 아래처럼 정리되었다.

### 4.1 결론
> **완전한 공개 교과서/교사용 자료 DB는 없다.**  
> 그러나 **MVP에 필요한 수준의 공개 메타데이터/기준/오픈 자산은 있다.**  
> 따라서 **교과서 그대로형 서비스는 어렵지만, 교육과정 정합형 생성 서비스는 가능**하다.

### 4.2 공개로 활용 가능한 축
1. **교육과정/성취기준 관련 공개 정보**
   - NCIC
   - 일부 에듀넷 공개 자료
   - 교육과정 연수자료

2. **교과서 메타데이터**
   - data.go의 `교육부_교과서 관련 정보`
   - 인정도서 목록 등

3. **오픈 라이선스 자산**
   - 공공누리 KOGL 0/1
   - 공유마당 CCL 중 CC BY 중심

4. **직접 제작 콘텐츠**
   - 문제 seed
   - 해설 seed
   - PPT 템플릿
   - 오개념 라벨
   - 재수업 자료 구조

### 4.3 공개로 활용이 어려운 축
- 로그인 뒤에만 보이는 평가도구 원문
- 교과서 본문 전체
- 교사용 폐쇄형 자료
- STAS 내부 도구/자료 원문
- NC/ND 등 상용 재가공이 어려운 자산
- 교과서/지도서 PDF 대량 수집

## 5. 오픈클로(OpenClaw) 검증 결과 해석 요약

사용자는 OpenClaw로 공개 자료 접근 가능성을 검증했고, report.md를 공유했다.  
그 보고서를 바탕으로 정리한 핵심은 아래와 같다.

### 5.1 NCIC
- 공개 접근은 가능
- 초등 국어/수학/영어/사회 성취기준 카드/인벤토리 수준은 확인 가능
- 하지만 이번 확인 범위에서는 **성취기준 코드 DB를 바로 완성할 정도의 구조화 데이터 확보는 불충분**
- 활용 방식:
  - 기준 인벤토리 확인
  - 성취기준 레퍼런스 링크 확보
  - 내부 기준 DB 작성 시 참고원

### 5.2 에듀넷
- 2022 개정 교육과정 연수자료 공개 페이지 접근 가능
- 초등 수학 PPT 등 일부 공개 다운로드 가능 확인
- 활용 방식:
  - 교과별 설명 구조 참고
  - 개념 설명/수업 흐름 참고
  - 내부 작성용 레퍼런스
- 주의:
  - 대량 자산 DB로 삼기보다 참고원으로 보는 편이 안전

### 5.3 data.go
- `교육부_교과서 관련정보_20191231` CSV 다운로드 가능 확인
- 이용허락범위 제한 없음
- 활용 가치가 매우 높음
- 활용 방식:
  - 교과서 메타데이터 DB
  - 서비스 첫 화면에서 `학년/과목/출판사/사용학년` 선택 UI 구축

### 5.4 KOGL(공공누리)
- 공개 검색/상세 확인 가능
- KOGL 0/1은 MVP 자산 후보
- KOGL 2/3/4는 차단 필요
- 활용 방식:
  - PPT용 이미지/시각자료/삽화 일부 수집
  - 메타데이터와 출처 저장

### 5.5 공유마당(CCL)
- CC BY 자산은 후보
- CC BY-SA는 보류
- NC/ND 포함은 차단
- 활용 방식:
  - 상업 사용 가능한 이미지/지도/시각자료 일부 확보

### 5.6 STAS
- 메뉴 구조는 공개 확인 가능
- 실제 도구/자료 활용은 로그인/권한 필요
- 활용 방식:
  - `평가유형`, `채점기준`, `수행평가 구조` 참고용
  - 직접 수집 원천 아님

### 5.7 오픈클로 보고서 기반 최종 해석
> **공개 자료만으로도 MVP는 부분적으로 가능하다.**  
> 다만 그것은 **공개 자료를 그대로 모아 서비스하는 방식이 아니라**,  
> `메타데이터 + 오픈 자산 + 자체 제작 콘텐츠` 방식이어야 한다.

## 6. 서비스 feasibility 최종 판단

### 6.1 가능한 것
현재 조건에서 아래는 가능하다고 판단됨.

- 학년/과목/출판사/학기 선택 UI
- 단원/영역 매핑표를 내부적으로 구축
- 해당 단원에 맞는 자체 문제 생성
- 자체 해설 생성
- 허용 자산을 넣은 수업용 PPT 생성
- 수준별 보충 문제 생성
- 오답/오개념 기반 보강 자료 생성

### 6.2 제한되는 것
아래는 제휴/권한/별도 확보 없이 어렵다.

- 교과서 본문 그대로 반영
- 출판사 수업자료 그대로 반영
- STAS 내부 자료 본문 대량 활용
- 교사용 폐쇄 자료를 기반으로 한 공용 DB 구축
- `완전 교과서 정합형`을 자동으로 보장하는 제품

### 6.3 따라서 제품은 어떻게 정의해야 하나
초기 MVP 정의는 아래가 맞다.

> **“교과서 복제형 서비스”가 아니라  
> “교육과정·교과서 메타데이터 기반의 정합형 생성 서비스”**

즉:
- `교과서 그대로`는 아님
- `교과서 흐름에 최대한 맞춘 내부 제작형`은 가능

## 7. 현재 가장 추천되는 MVP 방향

### 7.1 추천 MVP
> **교사용 AI 문제/해설/PPT 생성기**

입력:
- 학교급
- 학년
- 과목
- 출판사
- 학기
- 단원 또는 영역
- 학급 수준(쉬움/보통/도전)
- 수업 목적(도입/형성평가/복습/재수업)

출력:
- 형성평가 5~10문항
- 학생용 해설
- 오답 유도/오개념 기준 피드백
- 유사문항 재도전 세트
- 5~8장짜리 수업용 PPT 초안
- 교사용 보충 설명 포인트

### 7.2 왜 이 방향이 맞나
- 공개 교과서 원문이 없어도 만들 수 있음
- 교사의 pain과 직접 연결됨
- 학생 가치가 분명함
- 범용 AI와 차별화 가능
- 자료가 적어도 `좁은 범위 MVP`는 가능

## 8. 실제 데이터 파이프라인 초안

아래 구조가 현재 가장 현실적인 설계안으로 정리되었다.

### 8.1 `curriculum_standard`
역할:
- 교육과정 기준 DB

소스:
- NCIC
- 에듀넷 연수자료(참고용)

저장값:
- curriculum_year
- school_level
- subject
- grade_band
- area_or_unit
- standard_code (가능하면)
- source_url
- internal_summary
- concept_tags
- skill_verbs
- likely_misconceptions
- recommended_assessment_types

주의:
- 원문 전체 대량 저장보다
- 운영자가 읽고 **자체 요약/구조화**하는 방식이 더 안전

### 8.2 `textbook_catalog`
역할:
- 교과서 메타데이터 DB

소스:
- data.go 교과서 관련 정보
- 교육부 인정도서 목록(검수 보조)

저장값:
- school_level
- school_type
- title
- publisher
- author
- grade_use
- material_type
- curriculum_year

제품 활용:
- 교사가 `출판사/학기`를 선택하는 UI 구성

### 8.3 `open_asset`
역할:
- 합법적으로 쓸 수 있는 이미지/지도/삽화 자산 DB

소스:
- KOGL 0/1
- 공유마당 CC BY 중심

저장값:
- title
- origin_url
- creator_or_org
- asset_type
- keyword
- license_type
- commercial_ok
- derivative_ok
- attribution_text
- allow_for_mvp

운영 규칙:
- KOGL 2/3/4 차단
- NC/ND 차단
- BY-SA는 초기엔 보류 권장

### 8.4 `item_seed`
역할:
- 자체 제작 문항 seed

구성:
- 성취기준 1개당 최소 5종
  - 기초 확인
  - 오개념 진단
  - 적용
  - 서술형
  - 재도전

원칙:
- 외부 문제은행 수집이 아니라
- **직접 제작 + AI 변형**

### 8.5 `explanation_seed`
역할:
- 자체 제작 해설 seed

구성:
- 정답 이유
- 오답 이유
- 힌트 1
- 힌트 2
- 재도전 프롬프트
- 교사용 피드백 노트

## 9. Codex가 이어서 해야 할 일

Codex는 아래 순서로 이어받는 것이 좋다.

### 9.1 1단계 — 범위 축소
전 과목/전 학년으로 가지 말고 아래 중 하나만 선택:
- 초4 수학 1단원
- 초5 사회 1단원
- 또는 특수교사 맥락의 좁은 범위 1개

### 9.2 2단계 — 내부 기준 DB 스키마 확정
아래 테이블 스키마 구체화:
- curriculum_standard
- textbook_catalog
- open_asset
- item_seed
- explanation_seed

### 9.3 3단계 — 운영자용 입력 폼 설계
운영자가 다음을 직접 입력할 수 있게 화면 설계:
- 성취기준 요약
- 개념 태그
- 기능 태그
- 오개념 포인트
- 평가유형
- 문항 seed
- 해설 seed

### 9.4 4단계 — 교사용 생성 플로우 설계
예시:
1. 교사: 학년/과목/출판사/단원 선택
2. 시스템: 내부 단원 매핑 조회

## 10. 사용자 성향 / 커뮤니케이션 유의사항

사용자는 아래를 매우 중요하게 봄.

- 추상적인 말보다 **현실성**
- “될 것 같다”보다 **실제 가능한가**
- 공개 자료/권한/라이선스 구분
- 교사의 실제 pain 기반 아이디어
- 과장 없이 솔직한 가능/불가 구분
- 구체적인 실행 플로우
- 범용 AI와의 차이

또한 이전에 반말 사용을 불쾌하게 느꼈고, 이후에는 존댓말 유지가 중요하다.

## 12. Codex용 즉시 작업 제안

## 14. 마지막 판단

현재까지의 사실 기반 판단은 아래와 같다.

- 공개 자료만으로 **일부 MVP는 가능**
- 하지만 핵심 품질은 **자료 수집량**보다 **자체 제작한 기준 DB·문항·해설 구조**에 달림
- 아이스크림/T셀파처럼 방대한 자료형 플랫폼을 단기간에 복제하는 것은 비현실적
- 따라서 초기 전략은:
  - 좁은 과목/학년부터
  - 교사 pain이 강한 영역 선택
  - 자체 seed 자산 축적
  - 이후 교사 저자단/BYOC/제휴로 확장
