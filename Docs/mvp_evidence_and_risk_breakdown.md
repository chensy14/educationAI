## 2. 성취기준 정합형 vs 교과서 정합형

이 둘은 매우 다르다.

### 2.1 성취기준 정합형

의미:

- 학생이 무엇을 배워야 하는지
- 어떤 수준까지 다뤄야 하는지
- 어떤 표현과 활동이 허용되는지

를 교육과정 기준에 맞춰 만드는 방식이다.

장점:

- 공개 자료만으로도 어느 정도 가능
- 특정 교과서가 없어도 기본 학습 목표를 맞출 수 있음
- 생성형 MVP를 빨리 시험해볼 수 있음

단점:

- 특정 출판사 교과서의 단원명과 순서가 다를 수 있음
- 예시 문맥, 삽화, 활동 맥락은 교과서와 다를 수 있음
- 교사가 실제 수업에서 쓰는 표현과 조금 어긋날 수 있음

### 2.2 교과서 정합형

의미:

- 특정 출판사의 특정 학년 교과서 흐름
- 실제 단원명
- 본문 순서
- 활동 예시
- 익힘책/지도서 맥락

에 가깝게 맞추는 방식이다.

장점:

- 교사가 체감하는 실사용성이 높음
- "우리 반이 지금 배우는 것"과 훨씬 더 가깝다
- 수업 연결성이 자연스럽다

단점:

- 공개 자료만으로는 구현이 어렵다
- 교과서 원문, 지도서, 평가 자료 접근 문제가 생긴다
- 권한/제휴/BYOC 없이 고품질 구현이 힘들다

### 2.3 초등교사 관점에서 느껴지는 차이

교사는 보통 아래처럼 느낄 가능성이 크다.

- 성취기준 정합형:
  - "방향은 맞는데, 내가 쓰는 책이랑 완전히 같진 않네"
- 교과서 정합형:
  - "지금 우리 단원 흐름이랑 바로 이어지네"

즉 성취기준 정합형도 교육적으로는 의미가 있지만, 실사용 체감은 교과서 정합형이 더 높다.

## 3. 지금 리소스로 만들 수 있는 MVP의 안전선과 위험선

### 3.1 안전하게 말할 수 있는 것

아래는 지금 리소스로 비교적 안전하게 주장할 수 있다.

- 초등 수학의 일부 주제는 공개 연수자료 기반으로 작업 단위 테스트가 가능하다
- 성취기준을 참고한 문제/해설/PPT 초안 생성은 가능하다
- 좁은 주제에 대해 오개념 중심 형성평가 루프를 시험할 수 있다
- 교과서 메타데이터와 공개 자산을 붙여 MVP 구조를 설계할 수 있다
- `평면에서 점의 이동`, `원 그리기`, `막대그래프` 같은 주제는 샘플 생성 테스트 후보가 된다

### 3.2 조심해서 표현해야 하는 것

아래는 가능하더라도 표현을 조심해야 한다.

- "교과서 흐름과 유사한 결과물"
- "교사가 참고용으로 볼 수 있는 수준"
- "수업 초안 또는 활동 초안"

이 정도 표현은 가능하지만, 검수 전제라는 단서를 붙이는 편이 안전하다.

### 3.3 지금 말하면 위험한 주장

아래는 현재 리소스 기준으로 말하면 과장에 가깝다.

- 특정 출판사 교과서와 완전히 정합하다
- 실제 교실에서 바로 써도 무방하다
- 공신력 있는 평가 기준을 반영했다
- 학생 평가 결과를 신뢰도 높게 판정할 수 있다
- 교과서 없이도 현장 사용성 차이가 거의 없다

### 3.4 왜 간극이 생기나

교과서가 없으면 아래가 비게 된다.

- 단원명과 차시 흐름
- 실제 삽화와 활동 맥락
- 교과서 고유 표현
- 익힘책과 연결되는 연습 구조
- 교사용 지도서의 유의사항

그래서 성취기준은 맞더라도, 교사가 느끼기엔 `조금 뜬 결과물`이 될 수 있다.

## 4. 지금 단계에서 가장 현실적인 제품 정의

현재 리소스 기준으로 가장 현실적인 정의는 아래다.

> 교과서 복제형 도구가 아니라,  
> 공개 교육과정 자료를 참고해 `성취기준 정합형 수업 초안`을 생성하는 도구

이 정의 안에서는 충분히 MVP 테스트가 가능하다.

반대로 아래 정의는 아직 이르다.

> 특정 교과서를 그대로 따라가는 교사 실전 도구

이건 추가 자료 확보나 교사 검수 구조 없이는 무리다.

## 5. 실무적으로 다음에 해야 할 일

혼란을 줄이려면 다음 둘 중 하나가 꼭 필요하다.

1. 교사 검수 단계 추가
- 생성 결과를 현직 교사가 `실제 단원 흐름과 맞는지` 확인

2. BYOC 또는 교사 업로드 구조 추가
- 교사가 자기 교과서 단원명, 핵심 문장, 사진, 활동 흐름을 넣으면 정합도 상승

즉 지금 리소스만으로도 생성은 가능하지만, 실사용 정합도를 높이려면 `현장 입력`이 들어와야 한다.
